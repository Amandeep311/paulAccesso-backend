# Visitor Management System - Technical Documentation

## 1. Project Overview

This repository contains a Spring Boot backend for a Visitor Management System. It supports:
- OTP-based login for internal users
- Visitor registration and checkout
- Tag assignment and tag availability tracking
- Meeting status updates and email notifications
- Admin user management and role-based access
- Static frontend asset delivery from `src/main/resources/static`

The application is implemented as a Spring Boot service with JWT authentication, REST APIs, JPA persistence, and email-based OTP delivery.

## 2. Architecture

### 2.1 Logical Layers

- `controller`: exposes REST endpoints and handles HTTP requests
- `service`: contains business logic and transaction operations
- `repository`: persists data via Spring Data JPA
- `entity`: maps domain objects to database tables
- `dto`: transport objects for API requests/responses
- `security`: JWT authentication and user validation
- `config`: web, API prefix, and security configuration

### 2.2 Request Flow

1. Client calls a REST API endpoint.
2. `SecurityConfig` and `JwtAuthenticationFilter` enforce authorization.
3. Controller forwards request to service layer.
4. Service layer interacts with repositories and domain entities.
5. Responses are returned as DTOs.

## 4. Key Configuration

### 4.1 API Prefix

All controllers annotated with `@RestController` receive the `/api` prefix via `ApiPrefixConfig`.
Example: `@RequestMapping("/auth")` becomes `/api/auth`.

### 4.2 Security

`SecurityConfig` configures:
- CORS allow-all patterns
- CSRF disabled
- Stateless JWT sessions
- Public access for:
  - `/api/public/**`
  - `/api/auth/**`
  - `/api/users/employees`
  - `/api/visitors/*/meeting-status`
  - static frontend routes and assets
- Authentication required for `/api/**` other than allowed public paths

### 4.3 Database

`application.properties` contains MySQL settings and JPA options. The application uses:
- `spring.jpa.hibernate.ddl-auto=update`
- `spring.jpa.show-sql=true`
- `spring.sql.init.mode=always`
- `spring.sql.init.schema-locations=classpath:init.sql`

### 4.4 Email / OTP

The OTP system uses Spring Mail with SMTP settings from `application.properties`.

## 5. Authentication and Authorization

### 5.1 OTP login

- `POST /api/auth/send-otp`
  - Body: `{ "empId": "EMP001" }`
  - Sends OTP to the employee email
- `POST /api/auth/verify-otp`
  - Body: `{ "empId": "EMP001", "otp": "123456" }`
  - Returns `AuthResponse` with JWT tokens

### 5.2 JWT tokens

- Access and refresh tokens are generated by `JwtService`
- `JwtAuthenticationFilter` validates tokens for protected routes
- `POST /api/auth/logout` exists for client-side logout semantics, but the backend only logs the action and does not implement blacklist state

## 6. API Endpoints

### 6.1 Auth

- `POST /api/auth/send-otp`
  - Request: `LoginRequest`
  - Fields:
    - `empId` (String)
- `POST /api/auth/verify-otp`
  - Request: `VerifyOtpRequest`
  - Fields:
    - `empId` (String)
    - `otp` (String)
- `POST /api/auth/logout`
  - Header: `Authorization: Bearer <token>`

### 6.2 User Management

- `POST /api/users`
  - Creates new user (admin only)
  - Request: `UserRequest`
  - Fields:
    - `empId`, `name`, `email`, `mobile`, `designation`, `role`, `photo`
- `PUT /api/users/{id}`
  - Updates existing user (admin only)
- `DELETE /api/users/{id}`
  - Deletes user by ID (admin only)
- `GET /api/users`
  - Returns all users (admin only)
- `GET /api/users/employees`
  - Public list of employees
- `GET /api/users/receptionists`
  - Returns receptionists
- `GET /api/users/me`
  - Returns current authenticated user

### 6.3 Visitor Management

- `POST /api/visitors`
  - Register a visitor
  - Requires `Authorization` header
  - Request: `VisitorRequest`
  - Fields:
    - `name`, `mobile`, `email`, `company`, `address`, `personToMeetEmpId`, `purpose`, `photo`, `idProof`, `tagNumber`
- `POST /api/visitors/assign-tag`
  - Assigns a tag to an existing visitor
  - Request: `TagAssignmentRequest`
  - Fields: `visitorId`, `tagNumber`
- `GET /api/visitors/tags/available`
  - Returns available tags
- `PUT /api/visitors/{id}/checkout`
  - Checkout a visitor
  - Request: `CheckoutRequest`
  - Fields: `checkoutPhoto`
- `GET /api/visitors`
  - Returns visitor list for current user
- `GET /api/visitors/active`
  - Returns active visitors
- `GET /api/visitors/{id}`
  - Returns visitor by ID
- `GET /api/visitors/stats/today`
  - Returns count of today’s visitors
- `GET /api/visitors/stats/active`
  - Returns active visitor count
- `POST /api/visitors/{id}/resend-email`
  - Resend meeting request email for visitor

### 6.4 Tag Management

- `GET /api/tags/available`
  - Lists available tags
- `GET /api/tags`
  - Lists all tags
- `POST /api/tags/assign`
  - Assign tag to visitor

## 7. Data Model

### 7.1 User

- `id`, `empId`, `name`, `email`, `locationState`, `designation`, `department`, `company`, `role`, `photo`, `isActive`, `mobile`, `createdAt`, `updatedAt`
- Relationships:
  - One `User` can have many `Visitor` records

### 7.2 Visitor

- `id`, `name`, `mobile`, `email`, `company`, `address`, `personToMeet`, `purpose`, `photo`, `checkoutPhoto`, `idProof`, `tagNumber`, `checkInTime`, `checkOutTime`, `active`, `meetingStatus`, `user`

### 7.3 TagNumber

- `id`, `tagNumber`, `isAvailable`, `assignedToVisitorId`, `assignedAt`, `releasedAt`

### 7.4 Otp

- `id`, `email`, `otp`, `expiresAt`, `isUsed`

## 8. Database Initialization

The `init.sql` script creates:
- `users`
- `visitors`
- `tag_numbers`
- `otps`

It also inserts:
- admin user `ADMIN001`
- test admin user `TEST001`
- 50 tag entries

## 9. Build & Run

### 9.1 Requirements

- Java 17
- Maven
- MySQL database

### 9.2 Run locally

From the `visitor` module:

```bash
./mvnw clean package
./mvnw spring-boot:run
```

### 9.3 Packaging

The project uses Spring Boot Maven plugin. The artifact is built under `target/visitor-0.0.1-SNAPSHOT.jar`.

## 10. Deployment Notes

- Ensure `application.properties` values are replaced with environment-specific credentials.
- Externalize secrets such as database password, mail password, and JWT secret.
- Validate SMTP connectivity for OTP email delivery.
- Use a dedicated MySQL schema and secure connection strings.

## 11. Static Frontend

The backend serves static frontend assets from `src/main/resources/static`. This includes:
- `index.html`
- compiled React assets in `assets/`

The security config allows typical SPA route rendering for `/`, `/register`, `/log`, `/admin`, `/dashboard`, `/visitors`, and `/settings`.

## 12. Important Notes

- `logout` endpoint currently logs the event but does not implement token revocation.
- OTP verification supports hardcoded test OTP behavior in service logic.
- JWT authentication is required for most `/api/**` routes.
- CORS is currently permissive via `*` origin patterns.
